# OBED

Автономный бот для управления LP-позицией в Uniswap V3 в сети BNB Smart Chain (BSC).

Название проекта — потому что комиссии с позиции улетают на отдельный кошелёк, с которого
оплачивается обед. Когда-нибудь.

## Что делает бот

1. Раз в час (не чаще) проверяет, находится ли текущая цена пула внутри диапазона активной позиции.
2. Если позиции ещё нет — открывает её сам: берёт баланс кошелька, делает свап до пропорции
   50/50 по стоимости, минтит позицию с диапазоном `±RANGE_WIDTH_PCT` вокруг текущей цены.
3. Если цена вышла из диапазона — закрывает позицию (снимает ликвидность + комиссии),
   считает новый симметричный диапазон вокруг текущей цены, ребалансирует токены до 50/50
   и открывает новую позицию.
4. Если позиция в диапазоне, но накопленные комиссии (в пересчёте на стейблкоин) ≥ порога
   из конфига — забирает **только комиссии** (ликвидность не трогает), конвертирует
   нестейблкоин-часть в стейблкоин через своп и отправляет всё на отдельный адрес.
5. Состояние (`state/state.json`) и журнал действий (`state/actions.jsonl`) коммитятся
   обратно в репозиторий после каждого запуска — это и есть восстановление после рестарта:
   при следующем запуске бот просто читает state.json.

## Архитектура

```
.github/workflows/run.yml   # cron каждый час, выполняет main.py, коммитит state/
config.yaml                 # все параметры, без секретов
src/
  config.py                 # загрузка config.yaml + переменных окружения
  abis.py                   # минимальные ABI Uniswap V3 (mainnet BNB адреса)
  web3_client.py             # web3 + контракты + аккаунт из секрета
  math_utils.py              # tick <-> price конвертация, округление до tickSpacing
  state.py                   # чтение/запись state.json
  logger.py                  # логирование в stdout + JSONL
  position_manager.py        # open/close/check/collect
  swap.py                    # обмен через SwapRouter02
  main.py                    # точка входа, оркестрация одного "тика"
state/
  state.json                 # текущая позиция (commit в репо)
  actions.jsonl               # журнал всех действий (commit в репо)
```

## Настройка

1. Заполни `config.yaml`:
   - `pool.address` — адрес пула, который ты создал
   - `pool.token0` / `pool.token1` — адреса токенов пула **в том порядке, в котором их
     вернёт сам пул** (`token0()`/`token1()`), это не "как тебе удобнее"
   - `pool.fee_tier` — fee tier пула (500 / 2500 / 10000 и т.п., в сотых долях % * 100)
   - `fees.withdrawal_address` — адрес, куда улетают комиссии
   - `fees.stablecoin_address` — адрес USDT (BEP-20), уже указан по умолчанию
   - `position.range_width_pct` — ширина диапазона в %, симметрично от текущей цены

2. В Settings → Secrets and variables → Actions добавь:
   - `BOT_PRIVATE_KEY` — приватный ключ кошелька бота (без `0x` или с — код понимает оба варианта)

3. Закинь на адрес кошелька бота стартовый капитал (token0 + token1 в любой пропорции —
   бот сам ребалансирует до 50/50) и немного BNB на газ.

4. Запусти workflow вручную первый раз (`workflow_dispatch`) — бот откроет первую позицию.

## Важно

- Это рабочий каркас для реальных денег, но это всё ещё новый код. Перед заводом крупной
  суммы — обязательно прогони на маленькой сумме и проверь логи/транзакции в BscScan.
- Бот не угадывает направление цены и не является финансовым советом — он просто
  механически держит диапазон и собирает комиссии. Риск impermanent loss и риск самого
  смарт-контракта пула остаются полностью на тебе.
- Приватный ключ нигде в коде/конфиге/state — только секрет GitHub Actions.
